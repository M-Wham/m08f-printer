# AUR Package for m08f-printer ‚Äî Design

**Date:** 2026-06-18
**Status:** Approved
**Scope:** Add an Arch User Repository (AUR) package so Arch users can install
`m08f-printer` with an AUR helper (`yay -S m08f-printer`) instead of the manual
venv + `sed` + `lpadmin` steps in the README.

## Goal

A turnkey, versioned AUR package built from a tagged GitHub release. Installing it
must place everything a working printer needs: the Python package and `m08f` CLI,
the udev rule, the CUPS backend (wired to the system binary), the PPD, and the
license ‚Äî with no manual post-install editing.

## Decisions (locked)

- **Package style:** versioned source package built from a GitHub release tag
  (`v0.1.0` ‚Üí `archive/refs/tags/v0.1.0.tar.gz`). Not a `-git` VCS package, not
  PyPI-sourced. Reproducible, proper AUR hygiene, no PyPI dependency.
- **Canonical repo URL:** `https://github.com/M-Wham/m08f-printer`. The README and
  `pyproject.toml` currently point at `github.com/mwham/...` (a different,
  nonexistent account) and must be corrected as part of this work.
- **Files location in repo:** `packaging/aur/` (groups with future
  `packaging/deb/` and `packaging/install.sh` noted as planned in the README).
- **Checksum policy:** a real `sha256sum` of the release tarball. Never commit
  `SKIP` (that is only for VCS sources). This forces the release tag to exist
  before the PKGBUILD's checksum can be filled in.

## Deliverables

```
packaging/aur/
  PKGBUILD        # versioned, builds from GitHub release tarball
  .SRCINFO        # generated by `makepkg --printsrcinfo`
```

Plus doc corrections:
- `README.md` ‚Äî fix `mwham` ‚Üí `M-Wham` in URLs
- `pyproject.toml` ‚Äî fix `[project.urls]` Homepage/Repository/Issues

## PKGBUILD design

```bash
# Maintainer: mwham <mwams-mail@proton.me>
pkgname=m08f-printer
pkgver=0.1.0
pkgrel=1
pkgdesc="Linux driver/CLI/CUPS backend for the M08F A4 thermal printer (Phomemo / COLORWING / AIMO)"
arch=('any')                       # pure Python, no compiled extensions
url="https://github.com/M-Wham/m08f-printer"
license=('MIT')
depends=('python' 'python-pyserial' 'python-pillow' 'ghostscript' 'cups')
makedepends=('python-build' 'python-installer' 'python-wheel' 'python-setuptools')
source=("$pkgname-$pkgver.tar.gz::$url/archive/refs/tags/v$pkgver.tar.gz")
sha256sums=('<filled in after the tag is pushed and the tarball downloaded>')

build() {
    cd "$srcdir/$pkgname-$pkgver"
    python -m build --wheel --no-isolation
}

package() {
    cd "$srcdir/$pkgname-$pkgver"
    python -m installer --destdir="$pkgdir" dist/*.whl

    # udev rule (device access without sudo)
    install -Dm644 udev/99-m08f.rules \
        "$pkgdir/usr/lib/udev/rules.d/99-m08f.rules"

    # CUPS backend: substitute the build-time placeholder with the system binary
    # path, install mode 700 as CUPS requires for backends.
    install -dm755 "$pkgdir/usr/lib/cups/backend"
    sed 's|__M08F_BIN__|/usr/bin/m08f|' cups/m08f-backend \
        > "$pkgdir/usr/lib/cups/backend/m08f"
    chmod 700 "$pkgdir/usr/lib/cups/backend/m08f"

    # CUPS PPD
    install -Dm644 cups/m08f.ppd "$pkgdir/usr/share/cups/model/m08f.ppd"

    # License
    install -Dm644 LICENSE "$pkgdir/usr/share/licenses/$pkgname/LICENSE"
}
```

### Rationale for key choices

- **`arch=('any')`** ‚Äî the package is pure Python (no C extensions), so one
  artifact works on all architectures.
- **System Python deps, not pip** ‚Äî `python-pyserial` and `python-pillow` come
  from the Arch repos. AUR packages must not pip-install at build/runtime.
- **PEP 517 build/install** ‚Äî `python -m build` produces a wheel,
  `python -m installer` unpacks it into `$pkgdir`. This is the current Arch
  Python packaging guideline (no `setup.py install`).
- **CUPS backend path** ‚Äî a system install puts the CLI at `/usr/bin/m08f`, so the
  backend's `__M08F_BIN__` placeholder is replaced with that absolute path at
  package time. This removes the README's manual `sed` step entirely.
- **Backend mode 700** ‚Äî CUPS refuses to run backends that are group/world
  writable; 700 (root-owned in the installed system) is the safe default.

## Release / checksum workflow (sequencing)

The PKGBUILD checksum depends on a real release tarball, which depends on the tag
existing on GitHub. The agent executing the plan **cannot push to GitHub** from
the sandbox (SSH auth unavailable). Therefore:

**Precondition (human, before the plan runs):** tag `v0.1.0` is pushed to
`github.com/M-Wham/m08f-printer` so that
`https://github.com/M-Wham/m08f-printer/archive/refs/tags/v0.1.0.tar.gz`
resolves. The doc URL fixes ideally land before the tag, but if the tag is cut
first, the stale URLs inside the v0.1.0 source tarball are cosmetic ‚Äî the PKGBUILD
itself (what AUR consumers see) carries the correct URL, and the doc fix ships on
`main` for the next release.

With the tag present, the executing agent does everything else autonomously:
download the tarball ‚Üí compute sha256 ‚Üí write the PKGBUILD ‚Üí generate `.SRCINFO`
‚Üí test-build.

## Testing

- `makepkg` is available in this environment; `namcap` is not (treat `namcap` as
  an optional lint ‚Äî install if possible, otherwise skip with a note).
- Verify the checksum matches (`makepkg --verifysource` or `-g`).
- Build in the source dir with `makepkg -f` (it will pull `makedepends`); confirm
  it produces a `.pkg.tar.zst`.
- Inspect package contents (`tar tf <pkg>` or `bsdtar`) to confirm all five
  install targets land at the documented paths.
- `.SRCINFO` regenerated and matches the PKGBUILD.

## Out of scope

- Publishing to PyPI.
- A `-git` VCS variant.
- `.deb` package and install script (separately planned).
- Actually pushing the package to the AUR (the plan produces the files and a
  documented publish procedure; the human performs the AUR `git push`, which needs
  their AUR SSH key).
